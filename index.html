Why?

Existing bridges (pythonia, child_process) spawn a separate Python process and communicate over IPC. This adds latency, forces everything async, and loses type information.

node-api-python embeds CPython directly inside Node.js via a native addon. Python runs in the same process — function calls are just function calls.

Quick Start

Create a new project

$ npx node-api-python init my-project
$ cd my-project && npm install && npm start

Or add to an existing project

$ npm install node-api-python
const python = require('node-api-python')

// Import a Python module
const mod = python.import('./my_module')

// Sync — for scripts, CLIs
const result = mod.addSync(2, 3)  // 5

// Async — non-blocking, for servers
const prediction = await mod.predict(data)

// Zero-copy NumPy interop
const arr = new Float64Array([1.0, 2.0, 3.0])
const sum = mod.computeSync(arr)

Project templates

Start with a pre-configured template:

npx node-api-python init my-api --template express   # Express + Python ML

npx node-api-python init my-ml --template ml         # scikit-learn inference

Features

In-Process Execution

CPython runs inside Node.js. No subprocesses, no IPC, no serialization. Function calls are native calls.

Sync & Async API

mod.funcSync() for scripts, await mod.func() for servers. GIL-aware concurrency.

Zero-Copy NumPy

Float64Array ↔ numpy.ndarray via buffer protocol. No copying for large data.

TypeScript Generation

Auto-generate .d.ts from Python type hints. Full IDE autocomplete across the bridge.

Bidirectional Callbacks

Pass JS functions to Python. Events, progress callbacks, transforms — all work across the bridge.

Cross-Language Stack Traces

Python errors include pythonType, pythonTraceback, and combined JS+Python stacks.

Auto Type Conversion

Primitives, collections, BigInt, Date, Buffer, Set — all converted automatically both ways.

Prebuilt Binaries

npm install just works on Windows, macOS, Linux. No compiler needed.

API Reference

python.import(modulePath, options?)

Import a Python module. Returns a proxy with all functions.

const mod = python.import('./my_module')     // relative path
const np  = python.import('numpy')              // installed package

Calling functions

Every Python function is available in sync and async forms:

// Async (default) — returns Promise
const result = await mod.my_function(arg1, arg2)

// Sync — blocks, returns directly
const result = mod.my_functionSync(arg1, arg2)

TypeScript Type Generation

Auto-generate .d.ts files from Python type hints:

$ npx node-api-python generate-types ./my_module.py -o ./types/

Python

def compute(x: int, y: float) -> dict[str, float]:
    """Compute weighted values."""
    ...

Generated TypeScript

/** Compute weighted values. */
export function compute(x: number, y: number): Record<string, number>

Supports def, class, @dataclass, TypedDict, generics, Optional, Union, and docstrings as JSDoc.

Cross-Language Error Handling

When Python raises, you get a full stack trace across both languages:

try {
  mod.predictSync(data)
} catch (err) {
  err.pythonType       // "ValueError"
  err.pythonMessage    // "invalid input"
  err.pythonTraceback  // [{ file, line, function, source }]
  err.stack
  // ValueError: invalid input
  //     at predict (model.py:42) — raise ValueError("invalid input")
  //     --- Python/JS boundary ---
  // Error: ValueError: invalid input
  //     at Object.<anonymous> (server.ts:15:5)
}

TypeScript types: PythonError and PythonTracebackFrame are exported.

When to use what

  • node-api-python — when you need performance, sync calls, types, or NumPy interop
  • pythonia — when you want zero native dependencies and async-only is fine
  • child_process — when you just need to run a script once
